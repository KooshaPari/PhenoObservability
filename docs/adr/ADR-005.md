# ADR-005: PhenoObservability Log Management Strategy

## Status

**Accepted**

## Context

Logs remain essential for debugging and auditing. However, log volume can be overwhelming and expensive. A structured log management strategy balances comprehensiveness with cost and performance.

Key challenges:

1. **Volume**: Services generate massive log volume
2. **Structure**: Unstructured logs hard to query
3. **Retention**: How long to keep different log types
4. **Cost**: Log storage and indexing expensive
5. **Query Performance**: Fast log search at scale
6. **Compliance**: Audit log requirements

## Decision

We will implement a tiered log management system:

### 1. Log Structure Standards

**Rationale**: Structured logs enable efficient querying.

**Decision Details**:
- JSON format for all application logs
- Required fields: timestamp, level, message, service
- Recommended fields: trace_id, span_id, user_id
- Severity levels: DEBUG, INFO, WARN, ERROR, FATAL
- No multi-line logs; structured data instead
- Sensitive data automatic redaction
- Contextual fields over string concatenation

### 2. Log Levels and Sampling

**Rationale**: Not all logs are equally valuable.

**Decision Details**:
- DEBUG: Development only, never in production
- INFO: Normal operations, sampled if high volume
- WARN: Anomalies that don't affect service
- ERROR: Failures requiring attention
- FATAL: Service cannot continue
- High-volume services: 10% INFO log sampling
- ERROR/FATAL: Never sampled

### 3. Log Aggregation Pipeline

**Rationale**: Centralized collection for analysis.

**Decision Details**:
- Fluent Bit agents on all nodes
- Kafka for log buffering
- ClickHouse/Loki for log storage
- Grafana for log visualization
- Alert rules on log patterns
- Real-time and batch processing paths

### 4. Retention Strategy

**Rationale**: Cost optimization through tiered retention.

**Decision Details**:
- **Hot** (7 days): Full indexing, fast queries
- **Warm** (30 days): Compressed, slower queries
- **Cold** (1 year): Object storage, batch queries only
- **Audit** (7 years): Compliance logs, legal hold capability
- Automatic tiering based on age
- Custom retention per service possible

### 5. Log-Based Metrics

**Rationale**: Derive insights without storing all logs.

**Decision Details**:
- Extract metrics from logs at ingestion
- Error rate metrics from ERROR log counting
- Business metrics from structured log fields
- Log-derived SLOs where appropriate
- Pre-aggregation for high-cardinality dimensions
- Metric retention longer than log retention

## Consequences

### Positive

1. **Query Speed**: Structured logs fast to query
2. **Cost Control**: Sampling and tiering manage costs
3. **Debugging**: Correlation via trace IDs
4. **Compliance**: Audit log retention satisfied
5. **Insights**: Log-derived metrics long-term trends
6. **Standardization**: Consistent format across services

### Negative

1. **Migration**: Existing logs need conversion
2. **Training**: Developers must learn structured logging
3. **Overhead**: JSON parsing overhead
4. **Storage**: Still significant storage costs
5. **Complexity**: Multi-tier system to manage

### Neutral

1. **Vendor**: Can use CloudWatch, Splunk, or open source
2. **Schema**: Schema evolution for log formats
3. **Sampling**: Statistical accuracy tradeoffs

## Alternatives

### Alternative 1: Text Logs

**Considered**: Plain text logs with grep.

**Rejected Because**:
- Slow to query at scale
- No structure for aggregation
- Parsing overhead
- Not cloud-native

### Alternative 2: No Sampling

**Considered**: Keep 100% of all logs.

**Rejected Because**:
- Prohibitive cost
- Query performance degradation
- Most logs never queried

### Alternative 3: Application-Only Logs

**Considered**: No infrastructure/system logs.

**Rejected Because**:
- Misses important context
- Infrastructure issues invisible
- Incomplete debugging picture

### Alternative 4: Short Retention Only

**Considered**: Keep logs for only 7 days.

**Rejected Because**:
- Cannot investigate past issues
- Compliance requirements not met
- Trend analysis impossible
