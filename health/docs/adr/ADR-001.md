# ADR-001: Health Check Interface Design

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** health

---

## Context

The health library needs a standard interface for implementing health checks that can be registered and executed.

---

## Decision

We will use **interface-based health checks with standard result format**:

```
┌────────────────────────────────────────────────────────────────┐
│              Health Check Interface                             │
│                                                                 │
│  type Checker interface {                                      │
│      Name() string                                             │
│      Check(ctx context.Context) CheckResult                    │
│  }                                                             │
│                                                                 │
│  type CheckResult struct {                                     │
│      Name    string                                            │
│      Status  string  // "healthy", "degraded", "unhealthy"     │
│      Message string                                            │
│      Duration string                                           │
│      Details map[string]interface{}                            │
│  }                                                             │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

---

## Consequences

### Positive

- **Extensible**: Easy to implement custom checks
- **Standardized**: Common result format
