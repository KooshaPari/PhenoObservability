# ADR-003: Kubernetes Probe Handlers

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** health

---

## Context

Kubernetes requires specific HTTP endpoints for liveness and readiness probes.

---

## Decision

We will provide **standard handlers for Kubernetes probes**:

```
┌────────────────────────────────────────────────────────────────┐
│              Probe Handlers                                     │
│                                                                 │
│  Liveness: /health/live                                        │
│  - Returns 200 OK always                                        │
│  - Kubernetes restarts container if this fails                 │
│                                                                 │
│  Readiness: /health/ready                                      │
│  - Runs all registered health checks                           │
│  - Returns 200 if all healthy, 503 if any fail                 │
│  - Kubernetes removes from service endpoints                   │
│                                                                 │
│  Detailed: /health                                             │
│  - JSON response with all check details                        │
│  - For human debugging                                          │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

---

## Consequences

### Positive

- **Compatible**: Works with Kubernetes out of the box
- **Standard**: Industry standard endpoints
