# ADR-002: Concurrent Check Execution

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** health

---

## Context

Health checks may need to run concurrently to minimize overall check time.

---

## Decision

We will **run checks sequentially with individual timeouts**:

```
┌────────────────────────────────────────────────────────────────┐
│              Check Execution                                    │
│                                                                 │
│  Strategy: Sequential with timeout per check                   │
│                                                                 │
│  for _, checker := range checks {                              │
│      ctx, cancel := context.WithTimeout(ctx, timeout)         │
│      result := checker.Check(ctx)                             │
│      cancel()                                                  │
│      // Store result                                            │
│  }                                                             │
│                                                                 │
│  Rationale:                                                    │
│  - Simpler implementation                                      │
│  - Predictable resource usage                                  │
│  - Easier error handling                                       │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

---

## Consequences

### Positive

- **Simple**: No goroutine coordination needed
- **Predictable**: Fixed resource usage

### Negative

- **Slower**: Total time = sum of all checks
