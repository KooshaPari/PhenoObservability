# ADR-002: Dashboard Deployment Strategy

## Status
Accepted

## Context
Dashboards must be deployed consistently across environments with appropriate configuration for each context. We need a deployment approach that:
1. Integrates with existing CI/CD pipelines
2. Supports multiple environments (dev, staging, prod)
3. Enables rollback capabilities
4. Provides deployment verification
5. Handles Grafana API authentication

## Decision
Implement a GitOps-style deployment using:
1. Grafana HTTP API for direct deployment
2. CI pipeline stage for automated deployment
3. Environment-specific configuration via variables
4. Idempotent deployment (update if exists)
5. Pre-deployment validation

## Rationale

### Deployment Method Comparison

| Method | Speed | Rollback | Complexity | Choice |
|--------|-------|----------|------------|--------|
| API Direct | Fast | Version-based | Low | Yes |
| Provisioning | Medium | Restart required | Medium | Partial |
| ConfigMap | Slow | Kubernetes native | Medium | No |
| Operator | Fast | CRD-based | High | Future |

### API-Based Deployment

**Advantages**:
- Immediate application
- No file system requirements
- Works with managed Grafana
- Fine-grained error handling

**Implementation**:
```go
func DeployDashboard(client *GrafanaClient, dashboard Dashboard) error {
    payload := map[string]interface{}{
        "dashboard": dashboard,
        "overwrite": true,
        "message":   "Deployed via CI/CD",
    }
    
    resp, err := client.Post("/api/dashboards/db", payload)
    if err != nil {
        return fmt.Errorf("deploy failed: %w", err)
    }
    
    return nil
}
```

### GitOps Integration

**Pipeline Stage**:
```yaml
deploy-dashboards:
  stage: deploy
  script:
    - go run ./cmd/deploy --environment=$CI_ENVIRONMENT_NAME
  only:
    - main
    - tags
```

**Branch Strategy**:
- `main` → Staging deployment
- `tags/v*` → Production deployment
- Feature branches → Dev deployment (optional)

## Consequences

### Positive
- Fast deployment cycles
- Immediate feedback
- Environment flexibility
- No infrastructure dependencies

### Negative
- API credential management
- Rate limiting considerations
- No built-in rollback in Grafana
- Network dependency

### Mitigations
- Secure credential storage (CI secrets)
- Retry logic with backoff
- Version-based rollback strategy
- Local caching for resilience

## Implementation Notes

### Deployment Tool

```go
// Deployer handles dashboard deployment
type Deployer struct {
    client    *grafana.Client
    validator *Validator
    logger    *slog.Logger
}

func (d *Deployer) Deploy(ctx context.Context, env string) error {
    dashboards := generateDashboards(env)
    
    for _, dash := range dashboards {
        if err := d.validator.Validate(dash); err != nil {
            return fmt.Errorf("validation failed for %s: %w", dash.Title, err)
        }
        
        if err := d.client.UpsertDashboard(ctx, dash); err != nil {
            return fmt.Errorf("deploy failed for %s: %w", dash.Title, err)
        }
        
        d.logger.Info("dashboard deployed", "uid", dash.UID, "env", env)
    }
    
    return nil
}
```
