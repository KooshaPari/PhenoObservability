# ADR-002: Panic vs Return on Missing Logger

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** logctx

---

## Context

When retrieving a logger from context that doesn't exist, we must decide between returning nil, a default logger, or panicking.

---

## Decision

We will **panic if no logger is found in context**:

```
┌────────────────────────────────────────────────────────────────┐
│              Retrieval Behavior                                 │
│                                                                 │
│  func From(ctx context.Context) *slog.Logger {                 │
│      if logger, ok := ctx.Value(loggerKey).(*slog.Logger); ok {│
│          return logger                                         │
│      }                                                         │
│      panic("no logger found in context")                       │
│  }                                                             │
│                                                                 │
│  Rationale:                                                    │
│  - Programmer error to call From without WithLogger           │
│  - Fail fast during development                                │
│  - Forces proper initialization                                │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

---

## Consequences

### Positive

- **Fail fast**: Catches bugs early
- **Explicit**: Forces proper setup

### Negative

- **Runtime panic**: Must be caught in tests
