# ADR-001: Context Key Design

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** logctx

---

## Context

The logctx library needs a way to store and retrieve loggers from context using type-safe keys.

---

## Decision

We will use **unexported int type for context keys**:

```
┌────────────────────────────────────────────────────────────────┐
│              Context Key Design                                 │
│                                                                 │
│  type contextKey int                                           │
│                                                                 │
│  const (                                                       │
│      loggerKey contextKey = iota                               │
│  )                                                             │
│                                                                 │
│  Benefits:                                                     │
│  - Unexported: Other packages cannot collide                 │
│  - Type-safe: int type prevents string collisions              │
│  - Simple: No complex key generation                         │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

---

## Consequences

### Positive

- **Collision-free**: Unexported keys
- **Type-safe**: Distinct type

### Negative

- **No introspection**: Keys not visible externally
