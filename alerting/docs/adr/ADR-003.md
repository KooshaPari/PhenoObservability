# ADR-003: Notification Integration

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** alerting

---

## Context

Alerts need to be routed to various notification channels (PagerDuty, OpsGenie, webhooks). We need a strategy for integrating with external notification systems.

### Requirements

1. **Prometheus Compatible**: Generate Alertmanager configs
2. **Multi-channel**: Support multiple notification providers
3. **Severity Routing**: Route by alert severity
4. **Configurable**: Environment-specific routing

---

## Decision

We will use **Alertmanager configuration generation** as the primary integration method:

```
┌────────────────────────────────────────────────────────────────┐
│              Notification Flow                                  │
│                                                                 │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐        │
│  │  Alert      │    │ Alertmanager│    │  Notifier   │        │
│  │  Rules      │───▶│   Config    │───▶│  (Webhook)  │        │
│  └─────────────┘    └─────────────┘    └─────────────┘        │
│                              │                                  │
│                              ▼                                  │
│                    ┌─────────────────────┐                      │
│                    │   Routing Tree      │                      │
│                    │                     │                      │
│                    │   critical ───▶ PD  │                      │
│                    │   warning ─────▶ Slack│                      │
│                    │   info ────────▶ Log  │                      │
│                    └─────────────────────┘                      │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### Rationale

| Option | Pros | Cons | Verdict |
|--------|------|------|---------|
| **Alertmanager** | Industry standard, routing | Extra component | **SELECTED** |
| Direct webhooks | Simple | No routing logic | Partial |
| Custom notifier | Full control | Maintenance burden | Rejected |
| shoutrrr | Multi-channel | Go-only | Future |

### Implementation

```go
type PagerDutyConfig struct {
    APIKey         string            `yaml:"api_key"`
    ServiceID      string            `yaml:"service_id"`
    IntegrationKey string            `yaml:"integration_key"`
    PriorityMapping map[string]string `yaml:"priority_mapping"`
}

func GenerateAlertManagerConfig(pd PagerDutyConfig, og OpsGenieConfig) string {
    return `global:
  resolve_timeout: 5m

route:
  receiver: 'default'
  routes:
    - match:
        severity: critical
      receiver: 'critical'
      continue: true
    - match:
        severity: warning
      receiver: 'warning'

receivers:
  - name: 'critical'
    pagerduty_configs:
      - service_key: '` + pd.IntegrationKey + `'
`
}
```

---

## Consequences

### Positive

- **Standard integration**: Works with existing Prometheus setups
- **Rich routing**: Grouping, inhibition, silences
- **Reliable**: Alertmanager handles retries
- **Observable**: Built-in metrics

### Negative

- **Extra dependency**: Requires Alertmanager deployment
- **YAML generation**: Templating complexity
- **Learning curve**: Alertmanager configuration
