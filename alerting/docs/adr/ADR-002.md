# ADR-002: Threshold Management Strategy

## Status: Accepted

**Date:** 2026-04-05  
**Author:** Phenotype Engineering Team  
**Project:** alerting

---

## Context

Alert thresholds need to be configurable, validated, and organized by severity level. We need a strategy for managing threshold configurations across different environments and services.

### Requirements

1. **Hierarchical**: Global defaults with local overrides
2. **Typed**: Different types for different metrics
3. **Validated**: Prevent invalid configurations
4. **Documented**: Self-documenting threshold purposes

---

## Decision

We will use **struct-based thresholds with validation functions**:

```
┌────────────────────────────────────────────────────────────────┐
│              Threshold Architecture                             │
│                                                                 │
│  AlertThresholds                                                │
│  ├── ErrorRate          float64  (percentage)                  │
│  ├── P99LatencyMs       float64  (milliseconds)                │
│  ├── P95LatencyMs       float64  (milliseconds)                │
│  ├── MemoryUsagePercent float64  (0-100)                       │
│  ├── CPUUsagePercent    float64  (0-100)                       │
│  ├── DiskUsagePercent   float64  (0-100)                       │
│  ├── QueueDepth         int      (count)                       │
│  └── DBPoolUsagePercent float64  (0-100)                       │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### Rationale

| Option | Pros | Cons | Verdict |
|--------|------|------|---------|
| **Typed Struct** | Type-safe, validated | Less flexible | **SELECTED** |
| Map[string]float64 | Flexible | No validation, unclear units | Rejected |
| YAML config | Standard | Runtime errors | Partial |
| Environment vars | 12-factor | Limited structure | Rejected |

### Implementation

```go
type AlertThresholds struct {
    ErrorRate          float64 `yaml:"error_rate"`
    P99LatencyMs       float64 `yaml:"p99_latency_ms"`
    P95LatencyMs       float64 `yaml:"p95_latency_ms"`
    MemoryUsagePercent float64 `yaml:"memory_usage_percent"`
    CPUUsagePercent    float64 `yaml:"cpu_usage_percent"`
    DiskUsagePercent   float64 `yaml:"disk_usage_percent"`
    QueueDepth         int     `yaml:"queue_depth"`
    DBPoolUsagePercent float64 `yaml:"db_pool_usage_percent"`
}

func DefaultThresholds() AlertThresholds {
    return AlertThresholds{
        ErrorRate:          5.0,
        P99LatencyMs:       1000.0,
        MemoryUsagePercent: 85.0,
        // ...
    }
}

func ValidateThresholds(t AlertThresholds) []string {
    var errors []string
    
    if t.ErrorRate <= 0 || t.ErrorRate > 100 {
        errors = append(errors, "error_rate must be between 0 and 100")
    }
    
    return errors
}
```

---

## Consequences

### Positive

- **Type safety**: Wrong units caught at compile time
- **Validation**: Clear error messages for invalid configs
- **Defaults**: Sensible defaults for quick start
- **Documentation**: Field names document purpose

### Negative

- **Fixed schema**: Adding new thresholds requires code change
- **No dynamic updates**: Must redeploy for threshold changes
